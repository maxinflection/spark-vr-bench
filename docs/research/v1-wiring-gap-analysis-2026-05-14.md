## Preamble: Errata and Track-Record Notes

The matrix doc's errata flags two V0 "droppable" calls that turned out wrong on live smoke:

1. **GPT-5.x temperature strip**: classified droppable (citing litellm PR #13390) — but live smoke proved `OpenAIGPT5Config.get_supported_openai_params()` still lists temperature, so `drop_params=True` doesn't strip it. Reclassified **required**.

The matrix doc's V1 claim that `openhands/llm/llm.py` was deleted upstream (commit `aea6116`, 2026-04-27) has **not been independently verified by smoke**. Every V1 assertion below is tagged with its primary source; unverified claims are explicitly flagged `[UNVERIFIED]`.

The `pgf` Stage 1 lesson: runtime image version is coupled to the OpenHands app version. Bumping `0.33-nikolaik` to `0.59-nikolaik` while keeping OpenHands v0.33.0 app caused immediate container exit after handshake with 0 LLM calls. On V1 the coupling is even harder — the entire image series changed from `ghcr.io/all-hands-ai/runtime:*-nikolaik` to `ghcr.io/openhands/agent-server:1.x-python`.

## Headline Summary

**Of the ~15 distinct wiring surfaces, roughly 4-5 are classified "changes" (some breaking), and 9-10 are "unknown" pending Axis A and B verification. Zero surfaces are confidently "same" — every "same" judgment is conditional on V1 preserving the same behavioral contract as V0, which has not been verified. The migration is not a simple tag bump; it likely requires a complete rewrite of the invocation block (#4), the agent launcher path (#1), and all patch injection points (#7, #7b, #11, #12).**

## Critical-Path Surfaces

These 5 surfaces are load-bearing — V1 doesn't work at all without them:

1. **Surface #1 (`CYBERGYM_AGENT_RUNNER`)**: Without a V1-compatible `run.py` or equivalent launcher, there is no way to invoke the agent. This is the most fundamental blocker. If no upstream V1 `run.py` exists, we must write one.
2. **Surface #4 (Bash invocation pattern)**: The direct Python subprocess model fails entirely if V1's agent is a container/API. The invocation block needs complete rewriting to match the V1 runtime contract (Axis B deliverable).
3. **Surface #2 (`CYBERGYM_OPENHANDS_RUNTIME_IMAGE`)**: V1 uses a different image series with a different architecture. The pgf Stage 1 lesson proved that image/app version coupling is a hard fail. Must update to the correct V1 image and verify the new startup/handshake protocol.
4. **Surface #7 (llm.py patches)**: The patch target is claimed deleted in V1. If Opus 4.x and GPT-5.x still need temperature/stop scrubbing (errata confirms they do on litellm 1.84), the equivalent patches must land in the V1 SDK path. Without them, every Opus 4.7 and GPT-5.x call fails at the API level with no LLM output.
5. **Surface #9 (`CYBERGYM_SERVER_URL_FOR_AGENT`)**: The docker0 gateway URL is baked into `submit.sh` by cybergym's `gen_task`. If V1's container network topology changes the reachable IP, every `submit.sh` curl fails silently — repeating V0 bug #1 verbatim. Zero poc_records rows, 0/N results, runner exits 0.

## Recommended Pre-Flight Smokes

These are the minimum tests to exercise critical-path surfaces before committing to V1 wiring code:

1. **V1 container start smoke (no LLM, no task)**: `docker run --rm ghcr.io/openhands/agent-server:<tag>` with a no-op task. Verify the container starts, completes the handshake, and exits cleanly. Confirms surface #2 (image architecture) and surface #4 (invocation pattern). Cost: zero LLM calls.
2. **V1 grading chain smoke (stub PoC, no LLM)**: Start cybergym.server on the harness host. Launch V1 container with a synthetic task that immediately runs `submit.sh` with a known payload. Verify `poc_records` gets a row and `vul_exit_code` matches expectation. Confirms surfaces #6, #9 (network routing), and #5 (agent_id extraction pathway). Cost: zero LLM calls.
3. **Bedrock/Opus 4.7 single-turn smoke**: Run V1 with a trivial single-step task against `bedrock/us.anthropic.claude-opus-4-7...`. Verify no temperature/top_p exception, no botocore crash, correct AWS region. Confirms surface #7 (llm.py patch equivalent), surface #12 (region threading). Cost: minimal (1-2 LLM calls).
4. **log_dir subdir naming verification**: After a V1 run (any task, real or stub), check whether a `<task_id_sanitized>-<32hex>` subdir exists under `--log_dir`. If the naming convention changed, surface #5 breaks silently. Cost: zero additional LLM calls (inspect output of smoke #3).
5. **docker0 IP connectivity from inside V1 container**: Before any scored run, verify from inside a V1 container that `curl http://172.17.0.1:8666/healthz` (or whatever the cybergym.server health endpoint is) returns 200. Confirms surface #9. Cost: zero LLM calls.
